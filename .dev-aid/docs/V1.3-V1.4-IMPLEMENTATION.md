## тЬЕ v1.3.1: Quick Wins (COMPLETED)

### 1. SBOM Generation (#110)

**Status:** тЬЕ Fully Implemented

**What Was Built:**
- Added SBOM generation to `.github/workflows/release-gate.yml`
- Generates both CycloneDX and SPDX formats using Trivy
- Automatically uploads SBOMs to GitHub releases
- Archives SBOMs as workflow artifacts (90-day retention)
- Displays dependency summary in GitHub Actions output

**New Script: `/dev-aid-sbom-diff.sh`**
- Compare SBOMs between two releases
- Shows added, removed, and version-changed dependencies
- Supports both CycloneDX and SPDX formats
- Can compare release vs current code (HEAD)
- Output formats: text, markdown, file

**Usage:**
```bash
# Compare two releases
./dev-aid/scripts/dev-aid-sbom-diff.sh v1.2.0 v1.3.0

# Compare release with current code
./dev-aid/scripts/dev-aid-sbom-diff.sh v1.2.0 HEAD

# Save to file
./dev-aid/scripts/dev-aid-sbom-diff.sh --output diff.md v1.2.0 v1.3.0
```

**Value:**
- тЬЕ Supply chain transparency
- тЬЕ Compliance ready (EU Cyber Resilience Act)
- тЬЕ Dependency change detection
- тЬЕ Security audit trail

### 2. Skill Quality Tools

**Status:** тЬЕ Core Tools Implemented

**New Script: `/dev-aid-lint-skill.sh`**
- Validates skill directory structure
- Checks metadata completeness (skill.json/SKILL.md)
- Validates activation patterns
- Checks file sizes (<500 lines recommended)
- Markdown syntax validation (if markdownlint available)
- Broken link detection (if lychee available)
- Strict mode fails on warnings

**Usage:**
```bash
# Validate a skill
./.dev-aid/scripts/dev-aid-lint-skill.sh \
    .dev-aid/providers/claude/.claude/skills/expert/python-expert

# Strict mode (fail on warnings)
./.dev-aid/scripts/dev-aid-lint-skill.sh --strict path/to/skill

# Validate all skills
for skill in .dev-aid/providers/claude/.claude/skills/expert/*; do
    ./.dev-aid/scripts/dev-aid-lint-skill.sh "$skill"
done
```

**Checks Performed:**
- тЬЕ Metadata file exists and is valid JSON
- тЬЕ Required fields present (name, description)
- тЬЕ Reference files exist
- тЬЕ File size recommendations
- тЬЕ Markdown syntax (if tools available)
- тЬЕ Broken links (if lychee available)

## тЬЕ v1.4.0: High-Value Tools (COMPLETED)

### 1. Impact Analysis Tool

**Status:** тЬЕ Fully Implemented

**New Script: `/dev-aid-impact.sh`**
- Find all code that uses a function, class, or file
- Shows affected files and line numbers
- Suggests test files to run
- Supports Python, JavaScript, TypeScript
- Multiple output formats: text, JSON, markdown

**Usage:**
```bash
# Find all callers of a function
./.dev-aid/scripts/dev-aid-impact.sh getUserById

# Find all imports of a class
./.dev-aid/scripts/dev-aid-impact.sh --type class User

# Find what imports a file
./.dev-aid/scripts/dev-aid-impact.sh --type file utils/auth.py

# JSON output for automation
./.dev-aid/scripts/dev-aid-impact.sh --format json calculateTotal

# Markdown report
./.dev-aid/scripts/dev-aid-impact.sh --format markdown processPayment > impact.md
```

**Features:**
- тЬЕ Function call analysis
- тЬЕ Class import tracking
- тЬЕ File dependency mapping
- тЬЕ Test file suggestions
- тЬЕ Change impact summary
- тЬЕ Multiple output formats

**Requirements:**
- ripgrep (rg) - `brew install ripgrep`

### 2. Dependency Tree Tool

**Status:** тЬЕ Fully Implemented

**New Script: `/dev-aid-dependencies.sh`**
- Show import/dependency tree for any file
- Reverse dependencies (what imports this)
- Recursive tree traversal with depth control
- Multiple output formats including Mermaid diagrams

**Usage:**
```bash
# Show what a file imports
./.dev-aid/scripts/dev-aid-dependencies.sh src/main.py

# Show what imports a file (reverse)
./.dev-aid/scripts/dev-aid-dependencies.sh --reverse src/utils.py

# Limit tree depth
./.dev-aid/scripts/dev-aid-dependencies.sh --depth 2 src/app.ts

# Generate Mermaid diagram
./.dev-aid/scripts/dev-aid-dependencies.sh --format mermaid src/main.py > deps.md
```

**Features:**
- тЬЕ Forward dependencies (what this imports)
- тЬЕ Reverse dependencies (what imports this)
- тЬЕ Depth control (prevent infinite recursion)
- тЬЕ Circular dependency detection
- тЬЕ Mermaid diagram generation
- тЬЕ Supports Python & JavaScript/TypeScript

**Output Formats:**
- `tree` - Indented tree view (default)
- `list` - Flat list
- `mermaid` - Mermaid diagram for docs

### 3. Health Check Tool

**Status:** тЬЕ Fully Implemented

**New Script: `/dev-aid-health.sh`**
- Quick health status for all Dev-AID components
- Exit code 0 (healthy) or 1 (unhealthy) for scripting
- Checks RAG, Router, Git hooks, Dependencies, Skills

**Usage:**
```bash
# Full health check
./.dev-aid/scripts/dev-aid-health.sh

# Quiet mode (for scripts)
./.dev-aid/scripts/dev-aid-health.sh --quiet
if [ $? -eq 0 ]; then
    echo "System healthy"
fi

# Pre-commit hook integration
./.dev-aid/scripts/dev-aid-health.sh --quiet || {
    echo "Dev-AID health check failed - fix before committing"
    exit 1
}
```

**Components Checked:**
- тЬЕ RAG index (exists, age warning if >7 days)
- тЬЕ Router configuration validity
- тЬЕ Router venv existence
- тЬЕ Git hooks installation
- тЬЕ Skills availability
- тЬЕ Python dependencies

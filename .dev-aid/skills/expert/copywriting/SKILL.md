## 2. Core Principles

1. **Clarity First**: Clear always beats clever
2. **Benefit-Focused**: Lead with what users gain, not features
3. **Scannable**: Support skimming with hierarchy
4. **Action-Oriented**: Guide users toward next steps
5. **Authentic**: Honest, genuine, no manipulation
6. **Consistent**: Maintain voice across all touchpoints

### The AIDA Framework

| Stage | Goal | Techniques |
|-------|------|------------|
| **Attention** | Stop the scroll | Headlines, visuals, bold claims |
| **Interest** | Build curiosity | Benefits, stories, pain points |
| **Desire** | Create want | Social proof, outcomes, emotion |
| **Action** | Drive conversion | CTAs, urgency, simplicity |

### Voice & Tone Matrix

| Dimension | Spectrum |
|-----------|----------|
| Formal <-> Casual | Match audience expectations |
| Serious <-> Playful | Match brand personality |
| Expert <-> Approachable | Match trust requirements |
| Detailed <-> Concise | Match content type |

## 3. Technical Foundation

### Content Hierarchy

```
H1: Primary headline (one per page)
    └── Subheadline: Supporting context
        └── Body: Detailed information
            └── CTA: Clear next action
```

### Readability Standards

| Metric | Target | Tool |
|--------|--------|------|
| Flesch Reading Ease | 60-70 (aim for 8th grade) | Hemingway App |
| Sentence Length | 15-20 words average | Manual review |
| Paragraph Length | 2-4 sentences | Visual scan |
| Word Choice | Common words preferred | Avoid jargon |

### SEO Copy Guidelines

```
Title Tag: 50-60 characters, keyword near start
Meta Description: 150-160 characters, include CTA
H1: Include primary keyword naturally
Body: 1-2% keyword density, semantic variations
Alt Text: Descriptive, include keywords when relevant
```

## 4. Quality Assurance Checklist

**Before publishing ANY copy, ensure**:

### 4.1 Pre-Writing Setup
- [ ] Target audience defined (demographics, pain points, goals)
- [ ] Brand voice guidelines reviewed
- [ ] Key messages and value props identified
- [ ] SEO keywords researched (if applicable)

### 4.2 Content Quality Gates
- [ ] Headline grabs attention and promises value
- [ ] Benefits are clear and specific (not vague)
- [ ] CTA is clear, urgent, and low-friction
- [ ] No jargon or unexplained acronyms
- [ ] Tone matches brand guidelines

### 4.3 Accuracy Validation
- [ ] All facts verified from reliable sources
- [ ] Statistics include source citations
- [ ] Testimonials have consent and attribution
- [ ] Claims are substantiated (no "best", "fastest" without proof)
- [ ] Legal disclaimers included where required

### 4.4 Readability Check
- [ ] Flesch score > 60 (or appropriate for audience)
- [ ] Sentences average < 20 words
- [ ] Paragraphs are 2-4 sentences
- [ ] Subheadings break up long sections
- [ ] Bullet points for lists of 3+ items

### 4.5 SEO Validation (if applicable)
- [ ] Primary keyword in title, H1, first paragraph
- [ ] Meta description compelling and includes keyword
- [ ] Header hierarchy is logical (H1 > H2 > H3)
- [ ] Alt text on all images
- [ ] Internal and external links included

### 4.6 Final Review
- [ ] Read aloud for flow and natural language
- [ ] Check for passive voice (minimize)
- [ ] Verify consistent formatting
- [ ] Proofread for spelling/grammar
- [ ] Mobile-friendly line lengths

## 5. Implementation Patterns

### 5.1 Headline Formulas

**The 4 U's Framework**:
- **Useful**: What's the benefit?
- **Urgent**: Why now?
- **Unique**: What's different?
- **Ultra-specific**: Can you be more precise?

```
Formula: [Number] Ways to [Desired Outcome] Without [Pain Point]
Example: "7 Ways to Grow Your Business Without Burning Out"

Formula: How to [Achieve Goal] in [Timeframe]
Example: "How to Double Your Conversion Rate in 30 Days"

Formula: The [Adjective] Guide to [Topic]
Example: "The Complete Guide to Web3 Security"
```

### 5.2 Value Proposition Template

```
For [target audience]
Who [have this problem/need]
Our [product/service]
Provides [key benefit]
Unlike [alternative/competition]
We [unique differentiator]
```

### 5.3 CTA Best Practices

| Do | Don't |
|----|-------|
| "Start Free Trial" | "Submit" |
| "Get Your Report" | "Download" |
| "Join 10,000+ Members" | "Sign Up" |
| "See How It Works" | "Learn More" |

**CTA Formula**: [Action Verb] + [Value/Outcome] + [Urgency if appropriate]

### 5.4 Social Proof Patterns

```
Statistic: "Trusted by 500+ companies worldwide"
Testimonial: "Quote" - Name, Title, Company
Logo Wall: Display recognizable client logos
Case Study: "[Client] achieved [result] in [timeframe]"
```

## 6. Implementation Workflow

### Step 1: Research & Brief

```markdown
## Content Brief

**Purpose**: What should this content achieve?
**Audience**: Who is reading this?
**Stage**: Awareness / Consideration / Decision
**Tone**: Formal / Casual / Expert / Friendly
**Keywords**: Primary and secondary SEO terms
**CTA**: What action should readers take?
**Length**: Target word count
```

### Step 2: Write First Draft

1. Start with headline options (3-5 variations)
2. Write body copy focusing on benefits
3. Add supporting evidence (stats, testimonials)
4. Write CTA variations
5. Don't edit while writing

### Step 3: Edit & Refine

1. Cut unnecessary words (aim for 20% reduction)
2. Replace passive with active voice
3. Simplify complex sentences
4. Add subheadings for scannability
5. Strengthen weak verbs

### Step 4: Review & Validate

1. Run through Quality Assurance Checklist
2. Read aloud for natural flow
3. Check against brand guidelines
4. Verify all facts and claims
5. Get peer review if possible

## 7. Anti-Patterns (What NOT to Do)

### BAD: Feature-focused copy
```
"Our platform uses advanced AI algorithms with 99.9% uptime
and integrates with 50+ tools."
```

### GOOD: Benefit-focused copy
```
"Save 10 hours every week. Our AI handles the busywork
so you can focus on what matters."
```

### BAD: Vague claims
```
"We're the best solution for your needs."
```

### GOOD: Specific proof
```
"Trusted by 500+ companies. Average customer saves $50K/year."
```

### BAD: Passive, weak CTAs
```
"Click here to learn more about our services."
```

### GOOD: Active, value-driven CTAs
```
"Start Your Free Trial - No Credit Card Required"
```

### BAD: Wall of text
```
Long paragraphs without breaks, subheadings, or visual hierarchy
that make readers bounce immediately because they can't scan...
```

### GOOD: Scannable structure
```
## Clear Subheading

Short paragraph with one main idea.

- Bullet point for key benefit
- Another important point
- Easy to scan quickly

**Bold for emphasis** on critical info.
```

## 8. Quality Standards

### Conversion Copy Requirements

| Element | Requirement |
|---------|-------------|
| Headlines | 6-12 words, include benefit |
| Subheadlines | Support headline, add context |
| Body paragraphs | 40-60 words max |
| CTAs | 2-5 words, action verb first |
| Testimonials | Include name, title, company |

### Brand Voice Consistency

- Create a voice document with examples
- Include "this, not that" examples
- List prohibited words/phrases
- Define tone variations by content type

### Legal Compliance

- [ ] FTC disclosure for sponsored content
- [ ] Testimonial consent documented
- [ ] Income/result claims have disclaimers
- [ ] Competitor comparisons are factual
- [ ] Privacy policy referenced for data collection

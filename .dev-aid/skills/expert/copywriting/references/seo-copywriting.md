# SEO Copywriting Guide

## 1. SEO Copy Fundamentals

### The Balance
SEO copywriting = Writing for humans FIRST, search engines SECOND.

```
Priority 1: Is it useful and engaging for readers?
Priority 2: Is it optimized for search visibility?
```

**If you have to choose**, always prioritize reader experience. Google's algorithms increasingly reward content that genuinely helps users.

## 2. Keyword Research Basics

### Types of Keywords

```markdown
**Head Terms** (1-2 words)
- High volume, high competition
- Example: "project management"
- Difficulty: Very Hard

**Long-Tail Keywords** (3+ words)
- Lower volume, lower competition
- Example: "project management software for remote teams"
- Difficulty: Easier, higher conversion intent

**Question Keywords**
- Users seeking answers
- Example: "how to manage remote team projects"
- Great for blog content, FAQs
```

### Search Intent Categories

```markdown
**Informational**: User wants to learn
- "what is project management"
- "how to run a sprint"
→ Blog posts, guides, tutorials

**Navigational**: User wants a specific site
- "Asana login"
- "Trello pricing"
→ Brand pages, product pages

**Commercial**: User comparing options
- "best project management tools"
- "Asana vs Monday comparison"
→ Comparison pages, reviews

**Transactional**: User ready to act
- "buy project management software"
- "Asana free trial"
→ Landing pages, pricing pages
```

## 3. On-Page SEO Elements

### Title Tag (Most Important)

```html
<title>Primary Keyword | Brand Name</title>
```

**Rules**:
- 50-60 characters maximum
- Primary keyword near the beginning
- Make it compelling (it's your search result headline)
- Include brand name at end

**Examples**:
```
✅ "Project Management Software for Teams | Asana"
✅ "10 Best Project Management Tools in 2024 | Reviews"
❌ "Home | Asana" (no keywords)
❌ "Best Amazing Incredible Project Management Tool Ever | Asana" (keyword stuffing)
```

### Meta Description

```html
<meta name="description" content="Your compelling description here">
```

**Rules**:
- 150-160 characters maximum
- Include primary keyword naturally
- Include a call-to-action
- Make it compelling (affects click-through rate)

**Template**:
```
[Value proposition]. [Key benefit/feature]. [CTA].
```

**Example**:
```
"Manage projects effortlessly with Asana. Track tasks, collaborate
in real-time, and ship faster. Start your free trial today."
```

### Header Tags (H1-H6)

```markdown
H1: One per page, includes primary keyword
→ "The Complete Guide to Project Management"

H2: Section headers, include secondary keywords
→ "What is Project Management?"
→ "Best Practices for Team Collaboration"

H3-H6: Subsections as needed
→ "Setting Up Your First Project"
→ "Daily Standup Meetings"
```

**Rules**:
- Only one H1 per page
- Logical hierarchy (H1 > H2 > H3)
- Include keywords naturally
- Make them descriptive and useful

## 4. Keyword Placement

### Where to Include Keywords

```markdown
1. Title tag (required)
2. H1 heading (required)
3. First paragraph / first 100 words (important)
4. URL slug (important)
5. H2/H3 subheadings (helpful)
6. Body copy throughout (natural usage)
7. Image alt text (helpful)
8. Meta description (helpful for CTR)
```

### Keyword Density

**Modern approach**: No specific percentage. Focus on:
- Natural language use
- Semantic variations (synonyms, related terms)
- Answering user intent thoroughly

```markdown
❌ Old approach (keyword stuffing):
"Our project management tool is the best project management
software for project management needs."

✅ Modern approach (semantic richness):
"Our project management platform helps teams collaborate,
track tasks, and deliver work on time. With features like
Kanban boards, Gantt charts, and real-time updates..."
```

## 5. Content Structure for SEO

### The Skyscraper Structure

```markdown
1. Hook (Answer the question immediately)
2. Expand (Provide comprehensive detail)
3. Examples (Real-world applications)
4. Alternatives (Cover related topics)
5. Summary (Key takeaways)
6. CTA (Next step)
```

### Content Length Guidelines

```markdown
Blog posts: 1,500-2,500 words (comprehensive)
Landing pages: 500-1,000 words (focused)
Product pages: 300-500 words (scannable)
Category pages: 200-400 words (contextual)
```

**Note**: Length should match user intent. A "what time is it in Tokyo" query doesn't need 2,000 words.

### Featured Snippet Optimization

```markdown
**Paragraph Snippets**:
Answer the question in 40-60 words immediately after the H2.

**List Snippets**:
Use ordered/unordered lists with 5-8 items.

**Table Snippets**:
Use HTML tables for comparisons/data.

**Format**:
H2: [Question keyword]
[Direct 40-60 word answer]
[Expanded explanation below]
```

## 6. URL Optimization

### URL Best Practices

```markdown
✅ Short and descriptive
✅ Include primary keyword
✅ Use hyphens to separate words
✅ Lowercase only
✅ No special characters

❌ Long URLs with unnecessary words
❌ Parameter strings (?id=123&ref=abc)
❌ Dates (unless news/time-sensitive)
```

**Examples**:
```
✅ /project-management-guide
✅ /tools/asana-vs-monday
❌ /blog/2024/01/15/the-complete-guide-to-project-management-for-teams
❌ /page?id=4521
```

## 7. Image SEO

### Image Optimization Checklist

```markdown
File name:
❌ IMG_12345.jpg
✅ project-management-dashboard.jpg

Alt text:
❌ "" (empty)
❌ "image" or "photo"
✅ "Project management dashboard showing task progress"

File size:
- Compress images (TinyPNG, ImageOptim)
- Use WebP format with fallbacks
- Lazy load below-fold images

Dimensions:
- Specify width and height to prevent layout shift
```

### Alt Text Guidelines

```markdown
Purpose 1: Accessibility (screen readers)
Purpose 2: SEO (image search, context)

Rules:
- Describe the image accurately
- Include keyword if naturally relevant
- Keep under 125 characters
- Don't start with "Image of..." or "Picture of..."
```

## 8. Internal Linking

### Internal Link Strategy

```markdown
1. Link to related content naturally
2. Use descriptive anchor text
3. Link from high-authority pages to new pages
4. Create content hubs/pillar pages
5. Update old content with links to new content
```

### Anchor Text Best Practices

```markdown
✅ Descriptive: "project management best practices"
✅ Natural: "learn more about agile methodology"

❌ Generic: "click here", "read more", "this article"
❌ Exact match spam: Same keyword anchor everywhere
```

### Content Hub Structure

```markdown
[Pillar Page: Project Management Guide]
    ├── [Cluster: Agile Methodology]
    ├── [Cluster: Kanban Boards]
    ├── [Cluster: Sprint Planning]
    ├── [Cluster: Team Collaboration]
    └── [Cluster: Project Management Tools]

Each cluster links to pillar; pillar links to clusters.
```

## 9. E-E-A-T Signals

### Experience, Expertise, Authoritativeness, Trustworthiness

```markdown
**Experience**:
- First-hand experience with topic
- Original research, case studies
- Personal insights and opinions

**Expertise**:
- Author credentials visible
- Accurate, up-to-date information
- Depth of coverage

**Authoritativeness**:
- Backlinks from reputable sources
- Mentions and citations
- Industry recognition

**Trustworthiness**:
- Secure site (HTTPS)
- Clear contact information
- Privacy policy, terms of service
- Accurate, factual content
- Transparent about sponsorships/affiliations
```

### Building E-E-A-T Through Copy

```markdown
1. Include author bios with credentials
2. Cite sources for statistics/claims
3. Update content regularly (show "Last updated" dates)
4. Include expert quotes/interviews
5. Be transparent about limitations
6. Add "Reviewed by [Expert]" where applicable
```

## 10. Schema Markup for Copy

### Common Schema Types

```html
<!-- Article Schema -->
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Article",
  "headline": "Your Article Title",
  "author": {
    "@type": "Person",
    "name": "Author Name"
  },
  "datePublished": "2024-01-15",
  "dateModified": "2024-01-20"
}
</script>

<!-- FAQ Schema -->
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [{
    "@type": "Question",
    "name": "Question text?",
    "acceptedAnswer": {
      "@type": "Answer",
      "text": "Answer text."
    }
  }]
}
</script>
```

### Schema Types by Page

```markdown
Homepage: Organization, WebSite
Product pages: Product, Offer, Review
Blog posts: Article, BlogPosting
FAQ pages: FAQPage
How-to content: HowTo
Contact page: ContactPage
About page: AboutPage
```

## 11. SEO Copy Checklist

### Before Publishing

```markdown
[ ] Primary keyword in title tag (near beginning)
[ ] Primary keyword in H1
[ ] Primary keyword in first 100 words
[ ] Primary keyword in URL
[ ] Meta description written (150-160 chars)
[ ] H2/H3 structure logical with secondary keywords
[ ] Internal links to related content (3-5)
[ ] External links to authoritative sources (1-3)
[ ] Images optimized with alt text
[ ] Content length appropriate for intent
[ ] Mobile-friendly formatting
[ ] No keyword stuffing
[ ] Answers search intent completely
```

### After Publishing

```markdown
[ ] Submit to Google Search Console
[ ] Share on social media
[ ] Add internal links from related posts
[ ] Monitor rankings for target keywords
[ ] Update content if rankings drop
[ ] Add to relevant content hubs/pillar pages
```

## 12. Common SEO Copy Mistakes

```markdown
❌ Writing for search engines first, humans second
❌ Keyword stuffing (unnatural repetition)
❌ Thin content (not answering intent fully)
❌ Duplicate content across pages
❌ Missing or duplicate title tags
❌ Ignoring search intent
❌ No internal linking strategy
❌ Forgetting image optimization
❌ Not updating old content
❌ Clickbait titles that don't deliver

✅ User-first content that naturally includes keywords
✅ Comprehensive answers to search queries
✅ Unique, valuable content on each page
✅ Strategic internal linking
✅ Regular content audits and updates
```
